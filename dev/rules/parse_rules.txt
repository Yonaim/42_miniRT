
<우리의 파일 형식 규칙>

- Ambient light, Camera, Light는 subject를 따른다.
- Solid element의 경우 '<element_identifier> <고유 정보> [material] [texture]'와 같은 형식으로 들어온다.
	ex) sp 0,0,20 20 l s 255,0,0

	- element_identifier
		- 외부(?)식별자
			- C (camera)
			- A (ambient light)
			- L (point light)
		- 도형식별자:
			- sp (sphere)
			- pl (plane)
			- cy (cylinder)
			- co (cone)
			- bo (box)
	- material: 
		- l (lambertian) (default)
		- m (metal)
		- d (dielectric)
		- e (emmisive)
		- rand (random)
			- texture도 포함하여 랜덤으로 값을 설정한다.
			- 뒤에 다른 값이 나오면 바로 에러 처리
	- texture:
		- s (solid) (default)
			: s(r,g,b) (r, g, b 각각은 0 ~ 255 사이의 값을 가짐)
		- c (checker)
			: c((odd texture), (even texture))
				(재귀적으로 texture를 가짐)
				(일단은 texture 대신 rgb 컬러값을 받으며, 추후 수정 예정)
		- i (image)
			: i(image_path)
- #으로 시작하는 경우 주석이다.

<subject의 파일 형식 규칙>

- Each type of element can be separated by one or more line break(s).
- Each type of information from an element can be separated by one or more
space(s).
- Each type of element can be set in any order in the file.
- Elements which are defined by a capital letter can only be declared once in
the scene.

(example):
A 0.2 255,255,255
C -50,0,20 0,0,0 70
L -40,0,30 0.7 255,255,255
pl 0,0,0 0,1.0,0 255,0,225
sp 0,0,20 20 255,0,0
cy 50.0,0.0,20.6 0,0,1.0 14.2 21.42 10,0,255
cn 50.0,0.0,20.6 0,0,1.0 14.2 21.42 10,0,255
---------- 새로 추가 ---------
bo 30,0,0 0,20,10 255,255,255
(p_end1, p_end2, rgb)
